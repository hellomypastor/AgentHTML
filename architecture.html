Protocol Attributes

data-agent-action
发送给 agent 的 prompt 指令。唯一必填属性。按钮文本是用户看到的，action 是 agent 收到的。
data-agent-target
响应渲染位置。CSS 选择器、inline（触发元素之后）或 replace（替换父元素）。
data-agent-context
状态路径（逗号分隔），随 prompt 一起发送。特殊值 page 发送页面可见文本。
data-agent-bind
双向绑定输入控件值到状态 JSON 路径。用户输入实时同步到 window.agentHtml.state。
data-agent-render
响应渲染方式：html（默认）、text、patch（多区域 DOM 更新）、replace-state。

Skill Decision Flow

StepActionInputOutput
1 Scan 检查项目是否有现成的设计系统 tailwind.config / tokens.json / globals.css / :root vars 设计 token 或 null
2 Preset 根据内容语气选择 1 个预设 references/style-presets.md + artifact 用途 Preset A–E
3 Aware? 判断是否需要 agent-aware 交互 artifact 是否有可操作的离散项或可调参数 static 或 agent-aware
4 Category 加载分类参考（如有） references/category-*.md 分类特定模式或通用模式
5 Generate 生成单文件 .html 上述所有决策 + 用户 prompt self-contained .html artifact
